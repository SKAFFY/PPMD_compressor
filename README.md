# PPMC_compressor – реализация PPM-сжатия на Go (метод C)

Этот проект реализует **адаптивное арифметическое сжатие на основе частичного совпадения (Prediction by Partial Matching, PPM)** с использованием **escape-метода C (PPMC)**. Код включает:

- `ppmc_compress` – сжатие произвольного файла.
- `ppmc_decompress` – восстановление сжатого файла.
- `calgary_analysis` – анализ файлов Calgary Corpus: вычисление энтропии 0/1/2 порядка, реальный размер сжатых данных и средние байты на символ.

## Сборка

Убедитесь, что у вас установлен Go 1.18+ и Make.

```bash
make build
```

Исполняемые файлы появятся в папке `./bin`.

### Кросс-компиляция для Windows

Если вы работаете в Linux и хотите собрать `.exe` для Windows 10:

```bash
make build-windows
```

Бинарники появятся в `./bin/windows/`.

## Использование

### 1. Сжатие файла

```bash
./bin/ppmc_compress -f <входной_файл> -t <выходной_сжатый>
```

Дополнительные флаги:

- `-max-context-order int` – максимальный порядок контекста (по умолчанию 4)
- `-buffer-size int` – размер буфера записи в байтах (по умолчанию 4MB)
- `-progress` – показывать прогресс‑бар (по умолчанию true)

Пример:

```bash
./bin/ppmc_compress -f myfile.txt -t myfile.ppm -max-context-order 3
```

### 2. Распаковка

```bash
./bin/ppmc_decompress -f <сжатый_файл> -t <восстановленный_файл>
```

Пример:

```bash
./bin/ppmc_decompress -f myfile.ppm -t restored.txt
```

### 3. Анализ Calgary Corpus (14 файлов)

Сначала скачайте датасет (это сделает Make автоматически):

```bash
make calgary DIR=./test/test_dataset
```

Или с сохранением результатов в файл (папка `./test/reports`):

```bash
make calgary-save DIR=./test/test_dataset
```

Для ручного запуска:

```bash
./bin/calgary_analysis -dir ./test/test_dataset -output result.txt
```

Флаги анализатора:

- `-dir` – путь к папке с файлами Calgary Corpus (bib, book1, …, trans)
- `-output` – необязательный файл для сохранения таблицы (текстовый формат)
- `-max-order` – порядок контекста для PPM‑сжатия (по умолчанию 4)

## Краткое описание алгоритма

**PPM** предсказывает следующий символ на основе последних `order` символов (контекста). Если символ уже встречался в этом контексте, он кодируется арифметическим кодером. Если нет – кодируется специальный **escape‑символ**, и модель переходит к контексту на один символ короче. На самом нижнем уровне (порядок -1) используется равномерное распределение.

В этой реализации применяется **метод escape‑частоты C (PPMC)**: частота escape равна количеству уникальных символов в текущем контексте. Дерево контекстов хранит счётчики символов и позволяет быстро находить узлы. Арифметический кодер работает с 32‑битными регистрами.
